# 多源异构数据融合与统计检验分析平台 V1.0 开发实现任务书

## 1. 项目目标

本项目需要基于当前目录下两个 Word 文档中的功能说明与 Java 源码内容，开发一个可在本地运行的 Java Web 单体应用，用于完成“多源异构数据融合与统计检验分析平台 V1.0”的本地演示与软件著作权申报材料支撑。

V1.0 版本定位为本地演示版本，不要求接入真实 MySQL、真实外部 API 或生产级数据治理环境。系统应在本地启动后，通过浏览器访问 `http://localhost:8080`，使用默认账号 `admin / 123456` 登录，完成数据导入、清洗融合、统计分析、质量核验、报告导出、数据台账等主要流程演示。

## 2. 参考文档与源码定位

### 2.1 功能说明文档

参考文档：`多源异构数据融合与统计检验分析平台.docx`

该文档描述平台的主要功能范围，包括：

- 用户登录、注册、记住密码、忘记密码、退出确认。
- 数据源接入、文件批量导入、数据预览。
- 数据清洗、字段映射、标准化处理。
- 统计分析、质量核验、报告生成。
- 数据台账、通知公告。

### 2.2 源码文档

参考文档：`多源异构数据融合与统计检验分析平台源代码.docx`

源码文档中主要包含两个 Java 类，应按如下方式迁移和重构：

- `DuoyuanRongheChuli`：作为业务处理逻辑参考，拆分为文件导入、数据清洗、统计分析、质量检验、台账查询、报告导出等 Service。
- `DataFusionPlatform`：作为本地 SQLite 数据库、模拟数据和演示流程参考，迁移为 Spring Boot 启动初始化、本地数据库建表、演示数据生成和演示流程支撑逻辑。

源码文档未体现前后端分离结构，也未包含 Vue、React、Controller、Servlet 或独立前端工程。因此 V1.0 应按 Java 单体 Web 应用实现。

## 3. 技术方案

### 3.1 应用形态

- 应用类型：Java 单体 Web 应用。
- 后端框架：Spring Boot。
- 构建工具：Maven。
- 页面形态：Spring Boot 内置页面，采用 Thymeleaf 或项目内静态 HTML/CSS/JavaScript 实现。
- 本地数据库：SQLite。
- 默认数据库文件：`data_fusion.db`。
- 默认访问地址：`http://localhost:8080`。

### 3.2 数据与外部依赖

- 系统默认使用 SQLite 和内置模拟数据运行。
- 不要求用户本地安装 MySQL。
- 不要求接入真实外部 API。
- 数据库、API、Oracle 等数据源类型可以在页面中展示为数据源类型选项，但 V1.0 不实现真实连接。
- 文件导入需要真实支持 Excel 和 CSV 解析入库。

### 3.3 推荐依赖

Maven 项目建议包含以下依赖：

- `spring-boot-starter-web`
- `spring-boot-starter-thymeleaf` 或使用 Spring Boot 静态资源目录
- `spring-boot-starter-validation`
- `spring-boot-starter-jdbc`
- `sqlite-jdbc`
- `poi-ooxml`
- `commons-csv`
- `jackson-databind`
- `lombok`，可选

## 4. 用户与账号模块

### 4.1 登录

系统启动后，用户访问首页时应先进入登录页。用户输入账号和密码后，系统验证本地 SQLite 用户表中的账号信息。

默认内置管理员账号：

- 用户名：`admin`
- 密码：`123456`

### 4.2 注册

系统应支持本地账号注册。注册信息至少包括：

- 用户名
- 密码
- 确认密码
- 是否同意服务条款与隐私政策

注册时需要校验用户名唯一、密码和确认密码一致。

### 4.3 记住密码

系统应支持本地演示版“记住密码”功能。可通过浏览器 Cookie、LocalStorage 或本地会话标识保存登录便利信息。

### 4.4 忘记密码

忘记密码采用管理员重置方式实现。

功能要求：

- 普通用户在忘记密码页面看到“联系管理员重置密码”的提示。
- 管理员登录后，可在用户管理或账号维护入口中为本地用户重置密码。
- 密码重置仅基于本地 SQLite 数据库，不接入短信、邮箱或第三方认证。

### 4.5 退出确认

用户点击退出时，系统应弹出确认提示。用户确认后退出登录并返回登录页。

## 6. 功能模块要求

### 6.1 首页/仪表盘

首页应展示平台当前运行概况，包括：

- 数据源数量
- 已导入文件数量
- 总数据量
- 字段数量
- 基础质量检验通过率
- 最近导入记录
- 最近处理记录

首页数据来自 SQLite 中的导入批次、数据源、融合记录和质检结果。

### 6.2 数据源管理

数据源管理用于展示和维护演示数据源。

功能要求：

- 展示数据源列表。
- 支持新增演示数据源。
- 数据源类型包括 MySQL、API、Excel、CSV、Oracle。
- V1.0 中 MySQL、API、Oracle 只作为演示数据源类型保存连接信息，不要求真实连通。
- Excel、CSV 与文件导入模块联动。
- 支持刷新数据源状态。

数据源字段至少包括：

- 数据源名称
- 数据源类型
- 连接信息
- 状态
- 创建时间

### 6.3 文件导入与数据预览

文件导入是 V1.0 需要真实实现的核心功能。

支持格式：

- Excel `.xlsx`
- Excel `.xls`
- CSV

功能要求：

- 支持选择本地文件上传。
- 上传后解析文件表头和数据行。
- 展示字段名、字段类型推断、样例数据、总行数。
- 用户确认后，将数据写入 SQLite。
- 保存导入批次记录。
- 保存原始数据或结构化后的导入数据。
- 对解析失败、空文件、格式不支持等情况给出提示。

### 6.4 数据清洗/融合处理

V1.0 需要实现基础清洗和常见格式标准化，不实现自定义规则配置。

必须支持的处理能力：

- 字段映射
- 空值处理
- 首尾空格清理
- 重复数据去重
- 手机号格式标准化
- 邮箱格式标准化
- 日期格式标准化

处理结果应记录：

- 处理批次
- 源导入批次
- 处理前记录数
- 处理后记录数
- 清洗记录数
- 异常记录数
- 处理状态
- 处理时间

### 6.5 统计分析

V1.0 只实现基础指标，不实现趋势图、字段分布图、相关性分析或复杂多维分析。

统计指标包括：

- 总数据量
- 字段数
- 增量数据量
- 空值率
- 重复数
- 质量检验通过率

统计结果应基于已导入和已清洗的数据计算。

### 6.6 质量核验核查

质量核验用于对导入和清洗后的数据进行基础检查。

检查范围：

- 完整性检查：统计空值数量和空值率。
- 一致性检查：检查明显格式不一致的数据。
- 重复性检查：统计重复记录数量。
- 基础准确性检查：对手机号、邮箱、日期字段执行格式校验。

核验结果应展示：

- 检查项
- 检查说明
- 是否通过
- 问题数量
- 通过率
- 生成时间

### 6.7 报告导出

V1.0 只支持 Excel 和 CSV 导出。

支持导出内容：

- 当前页数据
- 全部数据
- 已筛选数据
- 统计结果
- 质量检验结果

不实现 PDF 导出。PDF 可作为后续版本规划。

导出记录应保存：

- 报告名称
- 导出格式
- 文件路径
- 导出用户
- 导出时间

### 6.8 数据台账

数据台账需要同时支持导入批次管理和业务分类筛选。

导入批次台账字段：

- 批次编号
- 文件名
- 文件类型
- 导入时间
- 记录数
- 状态
- 操作人

业务分类筛选包括：

- 用户
- 订单
- 产品
- 运营
- 客服
- 系统

功能要求：

- 支持按导入批次查看详情。
- 支持按业务分类筛选。
- 支持关键字搜索。
- 支持下载导出文件。

### 6.9 通知公告

V1.0 只展示系统内置公告，不实现管理员新增、编辑、删除公告。

内置公告类型包括：

- 维护提醒
- 安全更新
- 功能上线

公告字段包括：

- 标题
- 内容
- 类型
- 发布时间

### 11.3 文件导入验收

- 可上传 `.xlsx` 文件并预览。
